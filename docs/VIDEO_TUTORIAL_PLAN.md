# خطة سلسلة فيديوهات الشرح — ViewClass (فيو كلاس)

خطة إنتاج مكتملة لتسجيل فيديوهات تعليمية بالعربي تشرح النظام شاشة بشاشة،
مقسّمة **حسب نوع المستخدم** (5 مسارات) — كل مسار يشرح علاقة الدور بالنظام
وطريقة شغله بالترتيب.

- **المصدر المرجعي للميزات:** `docs/PROJECT_PLAN.md`
- **خريطة الشاشات الفعلية:** `resources/views/components/sidebar.blade.php` + `routes/web.php`
- **الحالة:** خطة هيكلية جاهزة للتنفيذ. كتابة السكربتات الكاملة = المرحلة التالية (انظر آخر القسم).

---

## 1. الهدف والجمهور

| البند | التفصيل |
|------|---------|
| **الهدف** | مكتبة أونبوردنج: أي مستخدم جديد يفتح مسار دوره ويتعلّم النظام من الصفر. |
| **اللغة** | عربي فصيح مبسّط، لهجة محايدة. |
| **الجمهور** | 5 أدوار: مدير النظام، مدير المدرسة/الوكيل، المعلم، الطالب، ولي الأمر. |
| **المخرَج** | لكل فيديو: هدف تعليمي واضح + شاشات محددة + مسار وصول + هنت عربي على الشاشة. |
| **مبدأ التقسيم** | كل دور يشوف مساره فقط. الشاشات المشتركة تتجمّع في **المسار 0**. |

---

## 2. معايير الإنتاج (تنطبق على كل الفيديوهات)

**المدة:** فيديو الشرح 4–8 دقائق. فيديو النظرة العامة (أول كل مسار) 2–3 دقائق.

**بنية كل فيديو (ثابتة):**
1. **مقدمة (10 ث):** شعار الأول + عنوان الفيديو + رقمه في المسار.
2. **أنا فين ومين أنا (15 ث):** الدور + الشاشة اللي هنشرحها انهاردة.
3. **مسار الوصول (Where-to-find):** نوضّح الطريق من القائمة الجانبية بالـ breadcrumb:
   > `القائمة الجانبية ← [القسم] ← [الشاشة]`
   مع تظليل العنصر في السايدبار وقت الضغط.
4. **الشرح خطوة بخطوة:** كل زر/حقل يتظلّل + هنت عربي جنبه.
5. **خلاصة (10 ث):** "دلوقتي تقدر…" + الفيديو اللي بعده.

**أسلوب الهنت العربي (On-screen callout):**
- أمر قصير مباشر يشاور على العنصر: «من هنا تقدر…»، «اضغط لـ…»، «⚠️ ملاحظة:…».
- صندوق مظلّل + سهم للعنصر. سطرين كحد أقصى لكل هنت.
- التحذيرات بأيقونة ⚠️ (زي قفل الخطة، منع الغش في الاختبار).

**التسمية:** `VC-[رقم المسار].[رقم الفيديو]-[اسم مختصر]`
مثال: `VC-3.6-exams-lifecycle`.

**الترتيب داخل المسار:** من النظرة العامة ← الأكثر استخدامًا يوميًا ← الأدوات المتقدمة.

**بيانات التجربة:** استخدم `developer@midade.com` وحسابات المعلم/الطالب/ولي الأمر
التجريبية (school_id=1). لا تسجّل ببيانات مستخدمين حقيقيين.

## 3. خريطة المستخدمين وعلاقاتهم (يُشرح مرئيًا في مقدمة كل مسار)

```
مدير النظام (منصّة الأول)
  │  يزوّد المدارس، يفعّل الخدمات، يشرف على كل المدارس
  ▼
مدير المدرسة / الوكيل ──────────────┐
  │ يدير مستخدمي مدرسته + العمليات    │ يقيّم أداء المعلمين
  │ التعليمية + التقارير              │ (زيارات صفية، أداء وظيفي)
  ▼                                   ▼
المعلم ──────────────► يدرّس، يحضّر، يرصد، يختبر، يتابع الحضور
  │  ينشئ محتوى (مواد/واجبات/اختبارات/فصول افتراضية/غرف نقاش)
  ▼
الطالب ──────────────► يحل الواجبات، يدخل الاختبارات، يتابع درجاته وحضوره
  ▲
  │ يتابعه
ولي الأمر ────────────► يشوف درجات/حضور/جدول أبنائه، يتواصل مع المعلمين
```

**العلاقات المفتاحية اللي لازم تتوضّح في الفيديوهات:**
- المحتوى بيتدفّق من **المعلم ← الطالب** (واجب/اختبار يُنشأ ثم يظهر للطالب).
- المتابعة بتتدفّق من **الطالب ← ولي الأمر** (نتيجة الطالب تظهر لوليّه).
- الإشراف بيتدفّق من **مدير المدرسة ← المعلم** (تقييم، اعتماد الخطط).
- التزويد بيتدفّق من **مدير النظام ← المدرسة** (إنشاء المدرسة والحسابات).

## 4. المسارات الخمسة

> صيغة كل فيديو: **العنوان · المدة · [الشاشات + مسار الوصول] · يغطّي · نموذج هنت.**

### 🟣 المسار 0 — شاشات مشتركة (كل الأدوار)

يُشاهده أي مستخدم مهما كان دوره، قبل مسار دوره.

**V0.1 — تسجيل الدخول وأول دخول** · 3 د
- الشاشات: `/login` — مسار الوصول: رابط النظام مباشرة.
- يغطّي: حقل البريد/المعرّف + كلمة المرور، «نسيت كلمة المرور»، التوجيه التلقائي لكل دور للوحته، تغيير اللغة (عربي/إنجليزي RTL).
- هنت: «اكتب بريدك أو اسم المستخدم هنا» · «⚠️ لو نسيت كلمة المرور اضغط هنا ويوصلك رابط».

**V0.2 — التنقّل في الواجهة** · 4 د
- الشاشات: الشِل (السايدبار + الهيدر) — مسار الوصول: أي صفحة بعد الدخول.
- يغطّي: أقسام القائمة الجانبية، البحث العام (`/search`)، قائمة الملف الشخصي، الإشعارات، طي/فرد السايدبار، معنى الأقسام (العمليات التعليمية / التواصل / النظام).
- هنت: «من هنا تتنقّل بين كل شاشات النظام» · «اضغط على اسمك لتعديل بياناتك أو الخروج».

**V0.3 — صندوق البريد الداخلي** · 5 د
- الشاشات: `/my/mailbox` (وارد/مُرسل/مسودات/أرشيف/مهملات) — مسار الوصول: `القائمة ← التواصل ← صندوق البريد`.
- يغطّي: قراءة رسالة، إنشاء بريد جديد، المرفقات، الأرشفة والحذف.
- هنت: «⚠️ الرسائل الرسمية بين الإدارة والمعلمين وأولياء الأمور بتوصل هنا».

**V0.4 — المواعيد والاستبيانات** · 5 د
- الشاشات: `/my/appointments` + `/my/surveys` — مسار الوصول: `القائمة ← التواصل ← المواعيد / الاستبيانات`.
- يغطّي: حجز موعد، متابعة مواعيدي، تعبئة استبيان مطلوب مني.
- هنت: «اختر اليوم والوقت المتاح واحجز» · «الاستبيانات المطلوبة منك بتظهر بعلامة».

### 🔴 المسار 1 — مدير النظام (Super-Admin) — مالك المنصّة

**نبذة الدور:** يملك منصّة «الأول»، يزوّد المدارس (Tenants)، يفعّل الخدمات،
ويشرف على كل المدارس. عنده كل صلاحيات مدير المدرسة زيادة.

**V1.1 — نظرة عامة + لوحة تحكم مدير النظام** · 3 د
- الشاشات: `dashboard` — `القائمة ← لوحة التحكم`.
- يغطّي: خريطة العلاقات (قسم 3)، مؤشرات المنصّة، فرق دوره عن مدير المدرسة.

**V1.2 — إدارة المدارس (تزويد Tenant جديد)** · 7 د
- الشاشات: `/admin/schools` (index/create/edit) — `القائمة ← النظام ← المدارس`.
- يغطّي: إنشاء مدرسة، الأقسام (بنين/بنات، ابتدائي/متوسط/ثانوي)، بيانات المدرسة والشعار، تفعيل/تعطيل مدرسة.
- هنت: «⚠️ كل مدرسة معزولة تمامًا — بياناتها ما تظهرش لغيرها» · «الشعار ده بيتحطّ كعلامة مائية على الشهادات».

**V1.3 — الأعوام الدراسية والفصول** · 5 د
- الشاشات: `academic_years`, `classes` — `القائمة ← النظام ← الأعوام الدراسية / الفصول`.
- يغطّي: إنشاء عام دراسي، تفعيل العام الحالي، إنشاء الصفوف والفصول والشُعب.
- هنت: «⚠️ العام النشط هو أساس كل الدرجات والحضور — خلي بالك وانت بتبدّله».

**V1.4 — خدمات الرسائل والإعدادات العامة** · 5 د
- الشاشات: `/admin/sms-services`, `/admin/settings` — `القائمة ← النظام ← خدمات الرسائل / الإعدادات`.
- يغطّي: تزويد رصيد/مزوّد الرسائل للمدارس، إعدادات المنصّة العامة.

**V1.5 — الإشراف عبر المدارس: البحث والتقارير والتصدير وسجل العمليات** · 6 د
- الشاشات: `/search` (كل المدارس), `/admin/reports`, `/admin/exports`, `/admin/activity-logs` — `القائمة ← النظام`.
- يغطّي: البحث في كل المدارس، التقارير المجمّعة، تصدير البيانات، تتبّع من عمل إيه (audit).

**V1.6 — الصيانة وأدوات النظام** · 4 د
- الشاشات: `/admin/maintenance` — `القائمة ← النظام ← الصيانة`.
- يغطّي: مسح الكاش، عمليات الصيانة، أدوات مدير النظام الحصرية.
- هنت: «⚠️ الأدوات دي حصرية لمدير النظام — استخدمها بحذر».

### 🔵 المسار 2 — مدير المدرسة / الوكيل (School-Admin) — يدير المدرسة

التقييم، الحضور، التواصل، والتقارير. أكبر مسار (مقسّم لثلاث حزم).

**V2.9 — إدارة الدرجات والتقارير** · 7 د

- هنت: «⚠️ تقييم "للبعض" بيتحسب نقاط إضافية بس، ولازم تصنّفه دعم أو إثراء».

- هنت: «⚠️ زيارة الدعم ما بتديش درجة — زيارة التقييم بتأثّر على نقاط المعلم».

ويُنشئ المحتوى اللي يوصل للطالب.

**V3.1 — نظرة عامة + لوحتي وجدولي وتقويمي** · 3 د

- يغطّي: علاقة المعلم بالمدير (تقييم/اعتماد) وبالطالب (محتوى)، جدولي الأسبوعي، تصدير PDF.
