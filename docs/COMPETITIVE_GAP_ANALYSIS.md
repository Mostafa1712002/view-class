## 2. الفجوات المؤكَّدة (grep = 0 نتيجة في كودنا)

### 🔴 فجوة 1 — العيادات (School Clinic) — حجم متوسط (~6 شاشات)
موديول طبي كامل عندهم، **مش موجود** عندنا (لا موديول `Clinic` في `app/Modules/`).
البنية الحقيقية (من المرجع مباشرة):
- **السجلات الطبية** (`/admin/Clinics/medicalReports`) — أعمدة: رقم الهوية، الاسم، المدرسة، الصف، الفصل، مرض مزمن، التحكم. زر «اضافة سجل طبي».
- **الأمراض** (`/diseases`) — كتالوج CRUD.
- **الأدوية** (`/medicines`) — كتالوج CRUD.
- **التطعيمات** (`/vaccines`) — كتالوج CRUD.
- **طلبات التحويل للعيادة** (`/requests`) — سير عمل إحالة طالب للعيادة.
- **التشخيصات** (`/diagnosises`) — تشخيص مرتبط بالسجل/الطلب.

### 🔴 فجوة 2 — الإرشاد الطلابي (Student Counseling) — حجم صغير (~2 شاشة)
`/admin/Coaching` عندهم — **إدارة برامج إرشاد**. **مش موجود** عندنا.
البنية الحقيقية (من المرجع مباشرة):
- جدول البرامج — أعمدة: اسم البرنامج، اسم المرشد، تاريخ البدء، تاريخ الانتهاء، عدد المستهدفين، حالة البرنامج، التحكم.
- أزرار: «اضافة برنامج»، تصفية، تخصيص الأعمدة، تصدير.
- إضافة برنامج = اسم + مرشد + فترة + الطلاب المستهدفون + حالة.

### 🟡 باج مكتشَف — المعلم ممنوع من الخطة الأسبوعية
`/manage/weekly-plans` بترجع **403 Forbidden** لدور المعلم، رغم إن المواصفات
(PROJECT_PLAN §2.4) بتقول المعلم يملك الخطة الأسبوعية. يحتاج مراجعة صلاحية/راوت.

## 5. حدود المقارنة الآلية (شفافية)

صفحات المرجع تُحمّل محتواها عبر AJAX بعد اختيار فلاتر، فالمسح الثابت لا يلتقط
المزايا الدقيقة داخل الشاشة. تدقيق «زر بزر» لكل موديول يحتاج تشغيل يدوي عميق
(اختيار فلاتر، فتح البُناة، مقارنة كل خيار). مردوده منخفض عند التطابق الحالي،
ويُفضَّل توجيهه لموديولات محددة عند الحاجة بدل مسح أعمى للـ 25 موديول.

**متاح عند الطلب:** «قارن [بنك الأسئلة / بناء الاختبار / رصد الدرجات …] بالتفصيل»
→ أدرِّل ذلك الموديول تحديدًا في المرجع وأقارنه شاشة بشاشة بنظيرنا.
